Planning CrewAI integration architecture (47 seconds)
Full Plan: CrewAI-based agent rebuild + dynamic .agent.md specs
1. Confirmed flaws to fix (from the ATAG.py/brain.py review)
#	Flaw	Where
F1	Every prompt (Planner, Browser, Critic, Coder) is a hardcoded Python string with .replace() substitution — no reuse, no easy iteration, playbooks baked into one giant blob	ATAG.py
F2	Memory retrieval is keyed on exact domain string (frontier.com ≠ spectrum.com), so a learned skill never transfers to a structurally similar but different site	memory_service.py
F3	"Success" is just "no error string in the last message" — no real verification against the goal, so bad outcomes can get written into memory and reinforced later	brain.py
F4	Critic only sees the raw error string and always does the same thing (rotate UA → retry same mission) — no classification of why it failed, so it can't tell "transient block" from "the whole plan is wrong for this site"	brain.py
F5	Planning is a single flat JSON list — no recursive decomposition for compound/novel tasks	ATAG.py
F6	Tool surface is fixed to "browser + Tavily search" even when a task doesn't need a browser at all	mcp_agent.py
2. Target architecture
Keep LangGraph as the top-level orchestrator (it already owns retries, browser session lifecycle, live streaming, CAPTCHA handling — rebuilding that in CrewAI would be wasted risk). Use CrewAI for the reasoning agents, each wrapped as one LangGraph node:

[understand_input]  (CrewAI: Planner agent)
        ↓
[generate_mission]  (CrewAI: Planner agent)
        ↓
   agent ⇄ tools     (LangGraph ReAct loop, unchanged execution engine —
   (existing loop)    but its system prompt now comes from browser.agent.md
        ↓             instead of a hardcoded static_sys_msg)
   on failure → [critic_task] (CrewAI: Browser agent, "recovery" task,
                 now returns a classified failure type, not just prose)
        ↓
[generate_script]   (CrewAI: Coder agent) — optional, unchanged trigger
        ↓
       END

CrewAI handles what to do next / why something failed / how to write the reproducer script. LangGraph keeps handling the actual tool-by-tool execution loop, streaming, and retries, because that part already works well and is tightly coupled to your UI.

3. .agent.md spec format (dynamic, no hardcoding)
Backend/src/agents/specs/
  planner.agent.md
  browser.agent.md
  coder.agent.md

Each file: YAML frontmatter + markdown body, e.g.:

---
name: browser_agent
role: Autonomous Browser Execution Specialist
goal: Complete the given mission on a live web page using available tools without triggering bot detection.
allowed_tools: [browser_click, browser_type, browser_scroll, browser_hover, browser_wait, browser_get_state, browser_navigate]
