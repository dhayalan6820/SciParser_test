You are a Senior UI/UX Engineer and React + Tailwind CSS expert.

Your task is to redesign and optimize the existing workflow/task execution component while preserving **100% of its current functionality and business logic**.

## Primary Objective

Transform the current component into a compact, modern, professional execution panel that occupies significantly less vertical space while remaining highly readable across all devices.

Do NOT rewrite the business logic unless absolutely necessary. Focus on UI, layout, responsiveness, spacing, and user experience.

---

## Design Goals

* Reduce the overall component height by 40–60%.
* Minimize unnecessary whitespace.
* Keep the interface clean and visually impressive.
* Improve information density without making the UI feel crowded.
* Ensure every element has a clear visual hierarchy.
* Follow modern SaaS dashboard design principles similar to Linear, GitHub, Vercel, Cursor, and Claude.

---

## Responsive Requirements

The UI must be fully optimized for every breakpoint.

Support:

* Mobile (320px–480px)
* Large Mobile (481px–640px)
* Tablet (641px–768px)
* Small Laptop (769px–1024px)
* Desktop (1025px–1440px)
* Large Desktop (1441px+)

The layout must adapt automatically without horizontal scrolling.

No content should overflow.

Icons, spacing, typography, badges, and buttons must scale appropriately for every screen size.

---

## Task Card Optimization

Reduce the size of every task card.

Reduce:

* Padding
* Margin
* Border radius
* Empty whitespace
* Icon spacing
* Vertical gaps

Keep the content readable while making the cards much more compact.

Each task should fit comfortably within a minimal height.

---

## Completed Task Behavior

When a task is completed:

* Collapse it automatically.
* Hide all completed subtasks.
* Show only a single summary row.

Example:

✓ Initialize Project          Completed

instead of displaying all completed subtasks.

Allow the user to expand it manually if needed.

---

## Active Task Behavior

Only the active task should remain expanded.

Display:

* Current task
* Current subtask
* Current progress
* AI reasoning (if available)

Everything else should remain collapsed.

This keeps the interface focused.

---

## Completed Workflow Behavior

When the entire workflow finishes:

Automatically collapse the entire execution panel.

Replace it with a compact summary card displaying:

* Workflow Completed
* Total Tasks
* Completed Tasks
* Total Time
* Success Status

Include a "View Execution" button to expand the full workflow history.

---

## Progress Section

Replace the large statistics section with a compact status bar.

Example:

✓ 8 Completed   ⟳ 1 Running   ○ 2 Pending   82%

Use a slim animated progress bar.

Reduce its vertical footprint.

---

## Typography

Optimize typography.

Use a clear visual hierarchy:

* Task Title → Medium weight
* Description → Smaller and muted
* Status → Compact badge
* Metadata → Minimal

Avoid oversized text.

---

## Icons

Reduce icon sizes slightly.

Use consistent sizing across all resolutions.

Keep animations smooth but subtle.

Avoid distracting effects.

---

## Animations

Use elegant Framer Motion animations.

Completed tasks:

* Fade out
* Collapse smoothly
* Slide upward

Workflow completion:

* Collapse with smooth animation
* Replace with summary card

Avoid excessive animation duration.

---

## Visual Design

Create a premium appearance.

Requirements:

* Modern glassmorphism
* Soft shadows
* Clean borders
* Proper spacing
* Consistent border radius
* Elegant gradients
* Professional color hierarchy

Avoid visual clutter.

---

## Performance

Optimize rendering.

* Prevent unnecessary re-renders.
* Use memoization where appropriate.
* Avoid layout shifts.
* Keep animations lightweight.

---

## Accessibility

Maintain accessibility.

* Keyboard navigation
* Proper focus states
* ARIA labels where necessary
* Sufficient color contrast

---

## Code Quality

* Keep the component modular.
* Preserve all existing functionality.
* Do not remove any business logic.
* Do not break existing APIs.
* Keep the code clean, maintainable, and production-ready.

Before finishing, review every breakpoint and ensure the component looks polished, balanced, and visually impressive on all screen sizes.

Do not stop after making small changes. Continue refining the layout until the design reaches production-quality standards with excellent responsiveness, compactness, and usability.
